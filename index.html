Django Rapido MINI V1.0

Lightweight Project Template Django 5.2 · REST APIs · Unfold Admin

A lightweight, production-ready Django project template designed for quick project setup. Perfect for smaller projects that don't need all the bells and whistles.

v1.0 Python 3.10+ Django 5.2 SQLite/PostgreSQL
01Overview

Core Features

Django 5.2 & DRF

Latest Django framework along with Django REST Framework to build REST APIs quickly.

Unfold Admin

Modern, beautiful Django admin interface with dark mode support and enhanced filtering.

Ready Database

SQLite by default, ready to use out of the box. Easy switch to PostgreSQL for production.

Auth & Docs

Token-based auth with custom user model. Auto-generated API documentation with DRF Spectacular.

02Quick Start

Get Started

Prerequisites: Python 3.10+ on Windows, macOS, or Linux.

One-command setup
bash
# 1. Clone the project
$ git clone <your-repo-url>
$ cd django-rapido-mini

# 2. Initialize the project
$ make init

# 3. Start the development server
$ make run
💡 What 'make init' does

Creates .env, generates SECRET_KEY, runs migrations, creates a superuser, and collects static files.

Default Credentials

After initialization, you can log in with username: admin and password: admin123. Change these in `.env` via DJANGO_SUPERUSER_USERNAME and DJANGO_SUPERUSER_PASSWORD.

05Database

Database Config

SQLite (Default)

Ready to use out of the box with no additional setup needed.

PostgreSQL (Production)

To switch to PostgreSQL, edit `.env` and uncomment the PostgreSQL section in project/settings/base.py. The `psycopg` library is already included.

env
# .env
DB_ENGINE=django.db.backends.postgresql
DB_NAME=project_db
DB_USER=your_user
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
06Environment

Environment Variables

Create a .env file (copy from .env.example):

env
# Django Settings
DJANGO_ENVIRONMENT=local
DEBUG=True
SECRET_KEY=your-secret-key-here
ALLOWED_HOSTS=localhost,127.0.0.1

# Admin Superuser
DJANGO_SUPERUSER_USERNAME=admin
DJANGO_SUPERUSER_EMAIL=admin@example.com
DJANGO_SUPERUSER_PASSWORD=admin123

11Customization

Project Customization

Creating Models
python
from django.db import models
from common.models import TimestampedModel

class MyModel(TimestampedModel):
    name = models.CharField(max_length=100)
    description = models.TextField()
Adding API Endpoints
  1. Create serializer in myapp/serializers.py
  2. Create view in myapp/views.py
  3. Add URLs in myapp/urls.py
  4. Include in main project/urls.py

13Troubleshooting

Common Issues

Migration Errors

Reset migrations (development only):

make db-reset

Static Files

If static files are not loading:

make collectstatic

Permission Denied

On Linux/Mac:

chmod +x manage.py

App Versions

This is the MINI version. For Celery, Redis, and more, use the full Django Rapido template.
